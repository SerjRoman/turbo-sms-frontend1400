Ваша задача реализовать основной функционал модуля Contact: 
    API запросы на получение списка контактов, 
    получение User по username, 
    создание контакта ( с помощью RTKq), 
    отображения списка контактов.

Необходимо подготовить необходимые типы для работы с API. Добавить константы для ссылки сервера и ссылки для картинок thumb

// Artem
GET Запрос /users/{username} - получение User по username. Отправляем запрос с динамическим параметром username и ожидаем сущность User без пароля

// Ilya
    GET Запрос /contacts/ - получение всех контактов. Отправляем запрос без параметров и ожидаем список контактов.
    GET Запрос /contacts/{id} - получение информации о контакте с его пользователем. Отправляем запрос с динамическим параметром id и ожидаем сущность Contact с полем contactUser(username, lastSeenAt)
    POST Запрос /contacts/ - создание контакта(мутация). Отправляем запрос с данными в body в формате FormData. В запрос принимаем: name, surname, contactUserId, avatar(null - берется аватар самого юзера),
//
Необходимо добавить необходимые свойства для avatar и объединить name и surname в localName(с пробелом) 
localName: name + " " + surname

Для правильной работы с кешем добавляем тег Contacts в baseApi. Даем этот тег для получения всех контактов и инвалдириуем тег при создании контакта.
Также нужно создать компонент ContactList в модуле Contact. Компонент использует FlatList для отображения массива контактов. Появление ошибки и загрузку нужно обрабатывать на основе Figma дизайна. \
Каждый item должен быть кликабельным и на текущий момент выводится просто “touched”. Для получения картинки используем ссылку на папку с thumb файлами бэкенда. http://localhost:8000/media/thumbnail/...
http://localhost:8000/media/thumbnail/${avatar}
